# Story 6.0.1: SYNKRA Framework Rebranding

## Status
**Approved**

## Story
**As a** AIOS framework maintainer,
**I want** to rebrand the framework from AIOS to SYNKRA with consistent naming across all components,
**so that** the framework has a unique identity that reflects its core value proposition of synchronizing 4 executor types and aligns with Pedro Valerio's task-first philosophy.

## Context & Motivation

### Why SYNKRA?
- **SYNKRA** = SYNchronization of executors in a KRA (flow/framework)
- Represents the 4 executor types: AGENTE (AI Auto), WORKER (AI Assist), HUMANO (Manual), CLONE (Cognitive)
- Aligns with Process Absolutism philosophy: "If it's not in SOURCE, it didn't happen"

### Naming Decisions (Roundtable Approved)

| Element | Current | New | Justification |
|---------|---------|-----|---------------|
| Framework | AIOS | **SYNKRA** | Sync + Flow |
| Master Agent | @aios-master | **@synk** | Derived from SYNKRA (SYN+K), identity noun |
| Knowledge System | *gold | **\*source** | Single source of truth |
| Core Folder | .aios-core | **.synkra-core** | Consistency |
| NPM Namespace | @aios-fullstack | **@synkra** | Simplification |
| Developer Agent | @aios-developer | **@synkra-dev** | Pattern consistency |

### Search Patterns for Validation
```bash

# Find all @aios-fullstack npm references

# Find folder references
```

### Backward Compatibility Strategy

1. **NPM Aliases**: `@aios-fullstack/*` packages redirect to `@synkra/*`
2. **Deprecation Warnings**: Old names emit console warning
3. **Timeline**: Support old names for 2 major versions
4. **Documentation**: Clear migration path documented

### Risks & Mitigations

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| Breaking changes in prod | Low | High | Aliases + testing |
| IDE incompatibility | Low | High | Test all 3 IDEs |
| Missed references | Medium | Medium | Comprehensive grep |
| User confusion | Medium | Low | Clear docs |

**4. Validation: Axioma Scoring**

**4.1. How Validation Works**
1. Extract artifact text
2. Scan for axiom keywords per level
3. Score each level (0-10) based on keyword density
4. Calculate overall score (weighted average)
5. Check minimum thresholds
6. Identify violations

**4.2. Scoring Algorithm**
```javascript
function validateAxiomas(artifact) {
  const scores = {
    existential: scoreLevel(artifact, EXISTENTIAL_KEYWORDS),
    epistemological: scoreLevel(artifact, EPISTEMOLOGICAL_KEYWORDS),
    social: scoreLevel(artifact, SOCIAL_KEYWORDS),
    operational: scoreLevel(artifact, OPERATIONAL_KEYWORDS)
  };

  const overall = (
    scores.existential * 0.3 +
    scores.epistemological * 0.25 +
    scores.social * 0.3 +
    scores.operational * 0.15
  );

  return {
    overall,
    breakdown: scores,
    passed: overall >= 7.0 && min(scores) >= 6.0
  };
}
```

**Pattern 3: Heuristic Pipeline**
```javascript
async function heuristicPipeline(context, heuristics, mindContext) {
  const results = [];

  for (const heuristicId of heuristics) {
    const heuristic = compileHeuristic(heuristicId, mindContext.config);
    const result = heuristic(context);

    // Early exit on veto
    if (result.veto) {
      return {
        status: 'VETOED',
        vetoTrigger: heuristicId,
        reason: result.vetoReason
      };
    }

    results.push({ heuristicId, result });
  }

  return { status: 'PASSED', results };
}
```

**6. Best Practices**

1. **Always handle both modes gracefully**
   - Check `if (mindContext)` before PV-specific logic
   - Provide Generic fallback
   - Log mode clearly

2. **Respect veto conditions**
   - Never override vetos without explicit user confirmation
   - Log veto reasons prominently
   - Provide actionable next steps

3. **Validate at the right time**
   - Validate outputs, not inputs
   - Early validation saves rework
   - Don't validate intermediate steps (performance)

4. **Cache compiled heuristics**
   - Compile once per session
   - Reuse across multiple calls
   - Clear cache on config changes

5. **Provide transparent reasoning**
   - Show PV scores and breakdowns
   - Explain why decisions were made
   - Link to underlying axiomas/heuristics

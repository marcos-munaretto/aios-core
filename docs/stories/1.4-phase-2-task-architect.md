PV_BS_001 is particularly relevant for architecture decisions:

**Example Scenario:**
```yaml

  clarity: 0.85  # Clear vision: "Real-time, event-driven architecture"
  description: "All systems react to events within seconds"

Market Signals:
  alignment: 0.3  # Current clients don't request real-time features

Heuristic Output:
  priority: HIGH  # (0.85 * 0.9) + (0.3 * 0.1) = 0.795
  recommendation: PROCEED
  reasoning: "End-state vision clarity (0.85) dominates market signals.
             Action directly enables future architecture."
```

**Without PV_BS_001 (Generic LLM):**
- Likely recommends: "Wait for market demand"
